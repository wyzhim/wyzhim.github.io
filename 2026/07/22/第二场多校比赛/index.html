M题√ 排列求值

题目描述

小羊对于一个图中，两个点可能不相连但是连通这件事感到很讶异。两个点相连指的是两个点之间
有边，而两个点连通指的是存在一条路径从一点出发到达另一点。
小羊给你一个n个结点的没有边的图，并让你给这个图上新增m条无向边。他希望你能最大化
(u,v) (u ̸= v)的对数，使得u,v不相连但是连通。图可能很大，所以你不需要完整的输出这个图的各
条边，只需输出这个对数的最大值即可。而这里(u,v),(v,u)被认为是相同的对。
注意图中不允许出现自环和重边。

输出描述:

对于每个测试样例，输出一个整数，表示连通但不相连的点对(u,v) (u̸=v)数量的最大值。

L题 懒得打

题目描述

小羊被分配了一个打乱长度为 $n$ 的排列的任务。一个排列就是一个长度为 $n$ 的包含 $1, 2, \dots, n$ 中的数各一次的序列。
小羊的领导会通过比较打乱前后数组的逆序对数量来评价小羊的工作。
也就是说，我们定义一个排列 $A = [A_1, A_2, \dots, A_n]$ 的逆序对数量为
$f(A) = \sum_{1 \le i < j \le n} [A_i > A_j]$
而打乱前后的排列分别是 $A$ 和 $A'$。那么，小羊的领导会计算
$g(A,A') = |f(A)-f(A')|$
来检查小羊的工作。领导希望这个数值越大越好。
但小羊是个懒人。他不想真的好好打乱数组，所以他决定使用一个长度为 $n$ 的排列 $p = [p_1, p_2, \dots, p_n]$，这样，当他工作中遇到一个排列 $A = [A_1, A_2, \dots, A_n]$ 的时候，他就可以提交
$f_p(A) = [A_{p_1}, A_{p_2}, \dots, A_{p_n}]$
作为他的工作成果。可以证明 $f_p(A)$ 也是一个排列。
小羊认为一个排列是 幸运的，当且仅当他使用上述的“打乱方法”的话，这个排列 $A$ 能最大化 $g(A,f_p(A))$。也就是说，对于任意一个长度为 $n$ 的排列 $A_0$，都有
$g(A,f_p(A)) \ge g(A_0,f_p(A_0))$。
那么有多少幸运的排列呢？鉴于答案可能很大，请将结果关于 $998244353$ 取模后输出。

注意，当 $condition$ 成立时，$[condition]$ 等于 $1$，否则等于 $0$。

输入格式

输入包含两行。

  • 第一行包含一个整数 $n$（$1 \le n \le 22$）— 排列的长度。
  • 第二行包含 $n$ 个整数 $p_1, p_2, \dots, p_n$（$1 \le p_i \le n$）— 小羊用于打乱 $A$ 的排列 $p$。
    保证对于所有 $1 \le i < j \le n$，有 $p_i \ne p_j$。

输出格式

输出一个整数，表示幸运排列的数量对 $998,244,353$ 取模后的结果。

题解核心推导（来自原题解）

逆序对贡献分析

对于原排列 A 中的一对位置 (i, j)（1 ≤ i < j ≤ n），打乱后它们变为 (p_i, p_j) 位置。

  • 若 p_i < p_j（即打乱后顺序不变），则这两个位置在打乱前后要么都形成逆序对，要么都不形成，贡献差为 0。
  • 若 p_i > p_j（即打乱后顺序反转），则：
    • 若 A_i > A_j，则打乱前是逆序对，打乱后不是，贡献为 +1；
    • 若 A_i < A_j，则打乱前不是逆序对，打乱后是，贡献为 -1。
      因此，只有 p 的逆序对会对差值产生贡献。
最大化绝对值

所有贡献的绝对值都是 1，若要让总和绝对值最大，应让所有贡献同向（全 +1 或全 -1）。
排列 A = [1, 2, ..., n] 恰好使所有逆序对（按 p 定义的）贡献为 +1，所以最大值可以达到。
因此，幸运排列 A 必须满足：对于 p 的每一个逆序对 (i, j)（i < j），要么 全部 满足 A_i < A_j，要么 全部 满足 A_i > A_j。

转化为拓扑排序计数

固定方向 A_i < A_j（另一种方向完全对称），我们可以建立一个有向图：
对于每个逆序对 (i, j)，添加一条有向边 i → j，表示 A_i 必须排在 A_j 前面。
那么满足条件的排列 A 的数量，就是这个有向图的 拓扑排序总数。
由于另一个方向（A_i > A_j）的图是原图的逆图，拓扑排序数量相同，所以最终答案为 2 × 拓扑排序数（注意：若无逆序对，则不存在任何限制，答案直接为 n!，不能乘以 2）。

状态压缩 DP 求拓扑序个数

由于 n ≤ 22，可以用二进制掩码表示已选节点集合。

  • 预处理每个节点 u 的前驱掩码 pre[u]（二进制第 i 位为 1 表示存在边 i → u）。
  • 令 dp[mask] 表示当前已选集合为 mask 时的拓扑序个数。
  • 转移：若节点 u 不在 mask 中，且 (pre[u] & mask) == pre[u]（即所有前驱都已选），则可将 u 加入，dp[mask | (1<<u)] += dp[mask]。
  • 最终 dp[(1<<n)-1] 即为拓扑序个数。

代码实现步骤（由简入繁）

第 1 步：读入数据
  • 读入整数 n。
  • 读入长度为 n 的排列 p（索引从 0 开始）。
第 2 步：判断是否存在逆序对

用双重循环枚举 i < j，若 p[i] > p[j]，则存在逆序对。用布尔变量 hasInv 记录。

第 3 步：处理无逆序对的情况（提前输出）
  • 若 !hasInv，则 p 本身升序，没有任何限制，任意 A 都是幸运的，答案为 n!（取模）。
  • 计算阶乘并输出，直接结束程序。
第 4 步：构建前驱掩码 pre
  • 初始化 vector<int> pre(n, 0)。
  • 再次遍历所有 i < j，若 p[i] > p[j]，则执行 pre[j] |= (1 << i)，表示 i 必须排在 j 前。
  • 注意：(1 << i) 是二进制第 i 位为 1 的数。
第 5 步：状压 DP 计算拓扑序个数
  • 令 full = (1 << n) - 1，表示所有节点都被选的掩码。
  • 创建 vector<long long> dp(1 << n, 0)，dp[0] = 1。
  • 外层循环 mask 从 0 到 full：
    • 若 dp[mask] == 0 则跳过。
    • 内层枚举节点 u（0 到 n-1）：
      • 若 mask 已包含 u（mask & (1<<u)），跳过。
      • 若 (pre[u] & mask) == pre[u]，则转移：
        • int nmask = mask | (1<<u)
        • dp[nmask] = (dp[nmask] + dp[mask]) % MOD
  • 循环结束后，topo = dp[full]。
第 6 步：计算最终答案
  • 因为有向图的两个方向（< 和 >）拓扑序数量相同，答案为 ans = (2 * topo) % MOD。
  • 注意：已经在第 3 步处理了 !hasInv 的情况，此处 hasInv 必为真。

关键二进制运算说明（供参考）

表达式 含义
1 << i 只有第 i 位为 1 的整数
mask & (1<<u) 检查 u 是否在集合 mask 中
`mask (1<<u)`
pre[u] & mask 取 pre[u] 和 mask 的公共位
(pre[u] & mask) == pre[u] 检查 pre[u] 中所有 1 位是否都在 mask 中（即前驱全部已选）

时间复杂度与空间

  • 时间复杂度：O(n * 2^n)，对于 n ≤ 22 约 9×10^7 次操作，可接受。
  • 空间复杂度：O(2^n)，约 4×10^6 个 long long（约 32 MB），合理。

注意事项（易错点）

  • 按位与 vs 逻辑与：条件中必须用 &，不能写成 &&。
  • 无逆序对时不要乘以 2：n! 已经包含了所有排列。
  • 移位溢出：建议使用 1LL << n 以防 n 较大（尽管本题 n≤22，用 int 安全）。
  • 模运算：所有加法、乘法都在 MOD 下进行。
