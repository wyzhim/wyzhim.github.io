四维dp

题目描述

小明过生日时，爸爸送给他一副乌龟棋作为礼物。
棋盘是一行 n 个格子，每个格子上有一个非负整数分数。第 1 格是起点，第 n 格是终点。
游戏中有 m 张爬行卡片，分为 4 种类型，分别标有数字 1, 2, 3, 4。使用一张卡片会让乌龟棋子向前移动对应的格子数，每张卡片只能使用一次。

游戏开始时，乌龟棋子自动获得起点格子的分数；此后每到达一个格子，就会获得该格子的分数。玩家的最终得分是所有经过格子的分数之和。

你需要决定卡片的使用顺序，使得最终得分最大。

输入格式

第 1 行：两个正整数 n, m，分别表示棋盘格数和卡片数。
第 2 行：n 个非负整数 a1, a2, …, an，表示每个格子的分数。
第 3 行：m 个整数，表示每张卡片上的数字（取值范围 1 ~ 4）。

数据保证：到达终点时，所有卡片刚好用完。

输出格式

一个整数，表示能得到的最大分数。

解题思路

状态设计
共有 4 种卡片，设其数量分别为 c1, c2, c3, c4。
定义四维 DP 数组：
dp[i][j][k][l] = 使用 i 张1步卡、j 张2步卡、k 张3步卡、l 张4步卡时，能获得的最大分数。
此时乌龟所在的位置为：
pos = 1 + i1 + j2 + k3 + l4

初始化
dp[0][0][0][0] = a[1] // 起点分数自动获得

状态转移
当前状态可以由少用一张某种卡片的状态转移而来：
dp[i][j][k][l] = a[pos] + max(
dp[i-1][j][k][l], // 最后一步用了1步卡
dp[i][j-1][k][l], // 最后一步用了2步卡
dp[i][j][k-1][l], // 最后一步用了3步卡
dp[i][j][k][l-1] // 最后一步用了4步卡
)
注意：当某种卡片数量为 0 时，对应的转移无效。

答案
最终答案为 dp[c1][c2][c3][c4]。

复杂度
时间复杂度：O(c1 * c2 * c3 * c4)，其中每种卡片的数量最多 31（因为 m ≤ 120），最坏约 31^4，完全可行。
空间复杂度：同样为 O(c1 * c2 * c3 * c4)。

常见错误分析

原先的一维 dp 状态存在以下问题：

  1. 状态不完整：同一个格子可能由多种不同的卡片组合到达，剩余卡片也不同。只保留最高得分会丢失其他可能在后续更优的状态，不满足最优子结构。
  2. 起点分数漏加：初始得分应为 a[1]。
  3. 状态覆盖错误：直接覆盖会丢失更优路径。
  4. 无法保证用完所有卡片：最终状态可能还有剩余卡片，但题目要求必须全部用完。

总结

本题的核心是四维动态规划，精确记录四种卡片的使用数量。
转移时，当前位置仅由卡片总数决定，无需单独存储位置。
保证所有卡片用完，就是取 dp[c1][c2][c3][c4]。
